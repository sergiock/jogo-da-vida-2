# Colaboradores no Github

> 16/04/2026 · 09:54 · terra · 03-capacidadesdocorpo · Agente social

## Agora
GitHub Pages para colaboradores funciona assim:

**Acesso de leitura** — qualquer pessoa pode ver o site publicado em `sergiock.github.io/jogo-da-vida` sem precisar de conta.

**Acesso para contribuir** — para alguém poder fazer commits no repositório:

```
github.com/sergiock/jogo-da-vida
→ Settings
→ Collaborators
→ Add people
→ digita o username ou email do colaborador
```

O colaborador recebe um convite por email. Depois de aceitar pode fazer push diretamente.

---

**Para colaboração mais controlada** — pull requests:

O colaborador faz um fork do repositório, edita na cópia dele, e propõe as mudanças via pull request. Você revisa e decide se aceita.

Isso é mais seguro — ninguém altera o repositório principal sem sua aprovação.

---

**Para o seu caso concreto** — se o seu filho vai colaborar tecnicamente, o fluxo mais simples é:

```
1  adicionar ele como collaborator
2  ele clona o repositório
3  trabalham em branches separados
4  merge quando estiver pronto
```

Qual dos dois modelos faz mais sentido — acesso direto ou pull requests?
